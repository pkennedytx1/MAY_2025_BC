1) What is the Back End?

The back end is the server-side portion of an application. It handles business logic, data storage, authentication, and communication with clients (front ends, mobile apps, other services). Think of it as the "brain" and the database as the "memory."

  • Server: Runs code that responds to requests (HTTP, WebSocket, etc.).
  • Database: Persists data (users, posts, transactions).
  • APIs (routes): Expose endpoints clients call (e.g. GET /users, POST /login).

2) Node + Express — the common combo

Node is a JavaScript runtime that runs on servers. Express is a minimal, flexible web framework for Node — it makes defining routes, middleware, and handlers easy.

Minimal Express server (what it looks like) 
// server.js
const express = require('express');
const app = express();
app.use(express.json()); // parse JSON bodies

app.get('/ping', (req, res) => res.json({status: 'pong'}));

app.listen(3000, () => console.log('Server listening on 3000'));

Key Express concepts:

  • Middleware: functions that run on requests (logging, auth, parsing).
  • Routes: endpoint handlers (GET, POST, PUT, DELETE).
  • Static files: express.static serves HTML/CSS/JS files.

3) Serving static files

Express can serve files from a folder. Useful for demo pages, frontend builds, or simple sites.

// serve 'public' folder
app.use(express.static('public'));

Put index.html and your bundle in /public, then visit http://localhost:3000/.

4) JSON Web Tokens (JWTs) — quick introduction

JWTs are compact tokens used to securely transmit information (often authentication) between parties. A token contains three base64 parts: header, payload (claims), and signature.

  • Use-cases: stateless auth, session tokens for APIs.
  • Important: Keep the signing secret safe. JWTs can be verified but not encrypted unless you use JWE.
Common JWT methods & flow
  1. User logs in with credentials → server verifies and issues a JWT.
  2. Client stores JWT (memory or secure storage) and sends it in Authorization: Bearer <token>.
  3. Server middleware verifies the signature and attaches the user to req.user.
// verify middleware sketch
const jwt = require('jsonwebtoken');
function auth(req,res,next){
  const header = req.headers.authorization || '';
  const token = header.split(' ')[1];
  if(!token) return res.status(401).send('no token');
  try{
    req.user = jwt.verify(token, process.env.JWT_SECRET);
    next();
  }catch(e){
    res.status(401).send('invalid token')
  }
}

5) Adding Express routes with JWT

Protect routes by placing auth middleware before route handlers. Example: app.get('/profile', auth, handler).

app.post('/login', async (req,res) => {
  // validate creds
  const token = jwt.sign({id: user.id}, process.env.JWT_SECRET, {expiresIn: '1h'});
  res.json({token});
});

6) Databases — Introduction

Databases store and retrieve data. Two broad categories: relational (SQL) and non-relational (NoSQL). Module 17 focuses on MongoDB (a document-based NoSQL DB).

7) Relational vs Non-Relational (practical comparison)

Relational (SQL)

  • Data stored in tables with rows and columns.
  • Strong schema, ACID transactions, joins across tables.
  • Use when data is structured and relationships are important (banking, accounting).
  • Common systems: PostgreSQL, MySQL, SQL Server.

Non-Relational (NoSQL)

  • Flexible schemas — documents, key-value, wide-column, or graph stores.
  • Scale horizontally more easily for some workloads; good for rapidly changing schemas and large datasets.
  • Use when you need flexible records or to store nested objects (e.g. product catalog, logs).
  • Common systems: MongoDB (document), Redis (key-value), Cassandra (wide-column), Neo4j (graph).
Example: modeling a blog post

Relational (normalized): posts table, users table, comments table, tags table + many-to-many join table for tags.

Non-relational (document): post document embeds author info and an array of comments inside the post document — fewer joins, easier reads for single-post API.

// MongoDB document example
{
  _id: ObjectId('...'),
  title: 'Hello',
  author: {id: 'u1', name: 'Alex'},
  content: '...text...',
  comments: [{user: 'u2', text: 'Nice!'}],
  tags: ['js','web']
}

8) MongoDB + Express — connecting

Typical stack: Express server uses a MongoDB driver (official) or an ODM like Mongoose to interact with a MongoDB instance.

// simple connection using the official driver
const {MongoClient} = require('mongodb');
const client = new MongoClient(process.env.MONGO_URL);
await client.connect();
const db = client.db('bad_bank');
const users = db.collection('users');

DAL (Data Abstraction Layer): keep DB access logic separated from route handlers — makes testing easier and keeps routes clean.

9) Connecting Back-End to Front-End

Back end exposes REST endpoints (or GraphQL). Front end calls these endpoints via fetch / XHR / axios. Use JSON as the data format.

// front-end fetch example
fetch('/api/profile', {headers:{Authorization: 'Bearer '+token}})
  .then(r => r.json())
  .then(data => console.log(data));

Common considerations: CORS, auth headers, environment-specific base URLs, and handling 401/403 responses gracefully.

11) Quick cheat sheet

  • Start server: node server.js or nodemon server.js
  • Env: JWT_SECRET, MONGO_URL
  • Common HTTP codes: 200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 500 Server Error
  • Dev tools: Postman, VSCode debugger, MongoDB Compass

Running MongoDB Locally

If you want to practice MongoDB on your own machine, here are two straightforward ways to get started without using Docker or MongoDB Atlas:

1️⃣ Install MongoDB Community Server Directly

  • Download the MongoDB Community Server installer for Windows, macOS, or Linux.
  • This installs the mongod server binary and the mongosh shell locally.
  • Start your server with mongod and connect to it using mongosh or MongoDB Compass.

2️⃣ Use Homebrew or a Package Manager

  • macOS: 
    brew tap mongodb/brew
            brew install mongodb-community@8.0
  • Linux: Add the official MongoDB repository for your distro and install via apt or yum.
  • This approach is simple and keeps your MongoDB version up-to-date.
