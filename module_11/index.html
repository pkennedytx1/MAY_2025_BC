Module 11: Introduction to JSX & React

üìå What is JSX?

  • JSX stands for JavaScript XML.
  • It allows you to write HTML inside JavaScript.
  • JSX makes it easier to create and visualize UI components.
  • Browsers can't read JSX directly ‚Äî it's compiled by tools like Babel into standard JavaScript.

üîç JSX Rules & Best Practices

  • Must return a single parent element: Wrap multiple elements in a div or React Fragment
  • All tags must be closed: Use self-closing tags like <img />
  • Use camelCase for attributes: className instead of class, onClick instead of onclick
  • JavaScript expressions in curly braces: {variable}, {2 + 2}, {user.name}
  • Comments: Use {/* This is a comment */}

üîç JSX Examples

// Basic JSX
const name = "React";
const element = <h1>Hello, {name}!</h1>;

// JSX with attributes
const element = <img src="logo.png" alt="Logo" />;

// JSX with multiple elements (must be wrapped)
const element = (
  <div>
    <h1>Title</h1>
    <p>Paragraph</p>
  </div>
);

// Using React Fragment (no extra div)
const element = (
  <>
    <h1>Title</h1>
    <p>Paragraph</p>
  </>
);

üîç Example: JSX vs. Regular JS

JSX:

const element = <h1>Hello, world!</h1>;

Compiled JS:

const element = React.createElement('h1', null, 'Hello, world!');

‚öõÔ∏è React Basics

  • React is a JavaScript library for building user interfaces.
  • It uses a component-based architecture.
  • Each component is a function (or class) that returns JSX.
  • React DOM renders these components into the actual HTML DOM.

üîç Basic Component Example

function Welcome() {
  return <h1>Welcome to React!</h1>;
}

‚öõÔ∏è React Components Deep Dive

  • Components are the building blocks of React applications.
  • They can be functional (using hooks) or class-based.
  • Components can be reused and composed together.
  • Each component should have a single responsibility.

üîç Functional Components

// Simple functional component
function Greeting() {
  return <h1>Hello, World!</h1>;
}

// Component with props
function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Using destructuring for cleaner code
function Greeting({ name, age }) {
  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>You are {age} years old.</p>
    </div>
  );
}

üîç Component Composition

  return (

üîÑ React State Management

  • State is data that can change over time in your component.
  • When state changes, React re-renders the component.
  • State is local to each component instance.
  • Use the useState hook to add state to functional components.

üîç useState Hook

import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
      <button onClick={() => setCount(count - 1)}>
        Decrement
      </button>
    </div>
  );
}

üîç Multiple State Variables

function UserProfile() {
  const [name, setName] = useState('');
  const [age, setAge] = useState(0);
  const [isActive, setIsActive] = useState(false);
  
  return (
    <div>
      <input 
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Enter name"
      />
      <input 
        type="number"
        value={age}
        onChange={(e) => setAge(Number(e.target.value))}
        placeholder="Enter age"
      />
      <label>
        <input 
          type="checkbox"
          checked={isActive}
          onChange={(e) => setIsActive(e.target.checked)}
        />
        Active
      </label>
      <p>Name: {name}, Age: {age}, Active: {isActive ? 'Yes' : 'No'}</p>
    </div>
  );
}

üîç State Updates

  • Direct updates: setCount(count + 1)
  • Functional updates: setCount(prevCount => prevCount + 1)
  • Object state: Use spread operator to update objects
  • Array state: Use spread operator or array methods

üîç Object State Example

function UserForm() {
  const [user, setUser] = useState({
    name: '',
    email: '',
    age: 0
  });
  
  const updateUser = (field, value) => {
    setUser(prevUser => ({
      ...prevUser,
      [field]: value
    }));
  };
  
  return (
    <div>
      <input 
        value={user.name}
        onChange={(e) => updateUser('name', e.target.value)}
        placeholder="Name"
      />
      <input 
        value={user.email}
        onChange={(e) => updateUser('email', e.target.value)}
        placeholder="Email"
      />
      <input 
        type="number"
        value={user.age}
        onChange={(e) => updateUser('age', Number(e.target.value))}
        placeholder="Age"
      />
      <pre>{JSON.stringify(user, null, 2)}</pre>
    </div>
  );
}

üéØ Event Handling

function EventExample() {
  const [message, setMessage] = useState('');
  
  const handleClick = () => {
    setMessage('Button clicked!');
  };
  
  const handleInputChange = (e) => {
    setMessage(e.target.value);
  };
  
  const handleSubmit = (e) => {
    e.preventDefault();
    alert('Form submitted: ' + message);
  };
  
  return (
    <div>
      <button onClick={handleClick}>Click me</button>
      <input onChange={handleInputChange} placeholder="Type something" />
      <form onSubmit={handleSubmit}>
        <input value={message} onChange={handleInputChange} />
        <button type="submit">Submit</button>
      </form>
      <p>Message: {message}</p>
    </div>
  );
}

‚úÖ React: Pros & Cons

üöÄ Advantages of React

  • Component Reusability: Build once, use everywhere. Components can be shared across projects.
  • Virtual DOM: Efficient updates by comparing virtual DOM with real DOM, minimizing expensive operations.
  • Large Ecosystem: Massive library of third-party components, tools, and libraries.
  • Strong Community: Extensive documentation, tutorials, and community support.
  • Backed by Meta: Facebook/Meta's backing ensures long-term support and development.
  • Cross-Platform: React Native allows mobile app development with the same skills.
  • Developer Tools: Excellent debugging tools like React DevTools.
  • SEO Friendly: Can be server-side rendered for better search engine optimization.

‚ö†Ô∏è Challenges & Limitations

  • Learning Curve: JSX, hooks, and React patterns can be overwhelming for beginners.
  • Rapid Evolution: Frequent updates and new features require continuous learning.
  • Tooling Complexity: Build tools, bundlers, and configuration can be complex.
  • Performance Overhead: Virtual DOM adds some overhead for simple applications.
  • State Management: Complex state management requires additional libraries (Redux, Zustand, etc.).
  • Bundle Size: Can lead to larger JavaScript bundles if not optimized properly.
  • SEO Challenges: Client-side rendering can impact SEO without proper setup.
